## 3. Wave 2 · Prototype：可运行 MVP

把 Specs 变成一个能跑通核心闭环的 MVP：

```text
角色创建 → AI 对话 → 场景识别 → 陪伴回应 → 轻量行动 → Check-in → 记忆摘要 → 桌面展示
```

### 功能优先级

**P0（必须）**
- 首页与入口：讲清"AI 角色互动 + 3D 桌面设备"，提供"开始体验"。
- 角色创建：可选类型、设名称/性格/语气/场景，内置 2–3 个预设角色（学习陪伴 / 原创二次元 / 数字宠物）。
- AI 对话：按设定回复、守语气，覆盖学习与日常陪伴场景。
- 学习陪伴主路径：低动力输入 → 陪伴回应 → 一个具体小行动 → 进入 10 分钟陪伴。
- Check-in：Done / Partly done / I got stuck，按选择生成简短反馈。
- 记忆摘要：保存本次互动、记录有效支持方式、展示"角色记住了什么"。
- 设备展示：呈现角色表情/动作/状态，说明设备如何与角色系统连接。

**P1（建议）**
- 基础语音输出（TTS 或预设语音）；角色状态标签（emotion_state 驱动表情/设备）；Creator Studio 上传原型；60–90 秒 Demo 视频。

**P2（后续）**
- 多角色皮肤；更稳的长期记忆库；真实语音输入；设备实时同步；订阅雏形；创作者数据看板。

### 技术重点

| 模块 | 实现方式 |
| --- | --- |
| 前端 | React / Next.js / Vite |

## 4. Wave 3 · Validation：用户测试与创作者试点

重点从"能不能跑"转到"用户和创作者是否真的需要"。要回答的是：用户是否愿意持续用、是否感到角色有性格有记忆、学习陪伴是否真帮到开始、设备是否明显提升存在感、创作者是否愿意接入、用户是否愿意付费。

### 测试对象

| 类型 | 人数 | 测试重点 |
| --- | --- | --- |
| ACGN / 二次元用户 | 10–20 | 角色互动与皮肤设定 |
| 学生 / 学习用户 | 10–20 | 学习陪伴与 10 分钟行动 |
| VTuber 粉丝 | 5–10 | 与创作者角色互动 |
| OC 爱好者 | 5–10 | 把自己的角色变 AI |
| 创作者 / 画师 / 独立游戏团队 | 3–5 | 是否愿意接入 Creator Studio |

### 测试流程

```text
招募 → 体验 Demo → 创建角色 → 一次对话 → 输入学习/陪伴场景
→ 体验 10 分钟陪伴 → 看记忆摘要 → 看设备 Demo → 问卷/访谈 → 迭代
```

| 指标 | 目标 |
| --- | --- |
| 完整体验人数 | 30–50 |
| 角色创建完成率 | ≥ 70% |
| 平均对话轮数 | ≥ 5 |
| 10 分钟行动启动率 | ≥ 40% |
| 重复使用意愿 | ≥ 50% |
| 设备正向反馈 | ≥ 60% |
| 初步付费意愿 | ≥ 20% |

### 迭代方向

根据测试结果，重点打磨：角色语气是否自然、记忆感是否到位；10 分钟行动是否太长太短、micro-action 是否够具体；设备是否够吸引、用户是否理解设备与 App 的关系；创作者最需要哪些控制能力与防偏离手段；以及最关键的商业化信号——用户愿意为什么付费、创作者接受哪种合作、设备适合销售/预售/高级体验。

## 8. 风险与应对

**定位过宽** —— 同时涉及 AI companion、ACGN、学习陪伴、创作者平台和设备。对外主线只讲"AI 角色互动系统 + 3D 桌面陪伴设备"，短期 MVP 聚焦学习陪伴与原创角色互动，Creator Studio 作后续扩展。

**技术过载** —— 对话、记忆、设备同步、Creator Studio 同时做会超出早期团队能力。Wave 2 只做 MVP，记忆先 Lite，设备先做展示型同步，Creator Studio 先做 Mockup，实时语音与设备同步后置。

**版权与 IP** —— 用户可能想上传未授权的知名或真人角色。初期只接原创/授权角色，Creator Studio 走白名单，配版权声明、审核与下架，不鼓励复刻。

**情绪依赖与未成年人** —— 陪伴产品可能被误解为替代现实关系。明确定位为日常陪伴与学习支持、非心理治疗或关系替代，设健康使用提醒，避免诱导依赖，对高风险表达给现实支持引导，对未成年用户设更严边界。

**硬件过早量产** —— 量产带来供应链、成本、售后、认证压力。当前设备只用于 Demo、验证与展示，核心价值仍是 AI 角色系统、记忆与创作者生态；先验证用户是否愿为桌面存在感付费，再考虑小规模测试或预售。

## 9. 最终愿景

Yorimi 想构建一个连接用户、虚拟角色、创作者 IP 与 3D 桌面设备的 AI 角色存在生态：用户拥有会长期记住自己、能陪学习工作生活的 AI companion；角色通过桌面设备出现在真实空间；VTuber、画师、独立游戏团队和原创 IP 能运营自己的 AI 角色；用户为语音、皮肤、记忆、限定互动和桌面体验付费。

不是又一个聊天机器人，也不是一个硬件展示装置，而是让虚拟角色真正拥有持续互动价值和现实存在感。

---

**一句话：** Wave 2 做出可运行 MVP，Wave 3 用真实用户和创作者验证需求，Wave 4 形成完整 Demo、数据、创作者试点与商业化方案，逐步把 Yorimi 从一份提案，长成 AI 角色互动系统与 3D 桌面陪伴设备结合的数字文化产品。

# 代码运行说明（Demo）

> 项目：Yorimi　·　阶段：初赛 Wave 1 / Wave 2 准备
> 在线 Demo：https://compagnon-eveil.vercel.app/ （想直接看效果，打开这个即可，无需本地运行）

Yorimi 的前身名为 **Compagnon Éveil**，所以当前线上 Demo 仍使用 `compagnon-eveil.vercel.app` 域名。本文件说明如何在本地跑起 Yorimi 当前的技术原型。这个 Demo 不是完整商业产品，目的是验证核心体验方向：角色创建 → AI 对话 → 状态识别 → 陪伴回应 → 轻量行动引导 → 记忆更新 → 3D 桌面展示。

---

## 1. 快速开始

> 前置：本机装好 Node.js 18 及以上和 Git。检查：`node -v`。

```powershell
git clone <你的仓库地址>
cd <项目文件夹>
npm install
Copy-Item .env.example .env
npm run dev
```

启动后打开：

```text
http://127.0.0.1:3017/nextstep-companion.html
```

`nextstep-companion.html` 是前身技术原型保留下来的兼容路由名；当前项目名称和提交身份是 Yorimi。

## 2. 配置 API Key

本仓库当前 live AI 路径使用 OpenAI runtime。在根目录复制 `.env.example` 为 `.env` 后填写：

```env
OPENAI_API_KEY=填这里
USE_MOCK_AI=false
```

未配置 `OPENAI_API_KEY` 时，系统会进入 schema-compatible mock fallback，用于展示接口结构、前端渲染和降级逻辑；真实角色对话质量与学习陪伴效果以 OpenAI runtime 为准。

## 3. 推荐体验路径

第一次上手，按这个顺序走一遍最顺：

1. **首页** —— 看产品定位是否一眼说清：AI 角色互动 + 3D 桌面存在。
2. **创建角色** —— 填名称、类型、性格、语气、使用场景。类型可选学习陪伴 / 原创二次元 / VTuber / 独立游戏角色 / 数字宠物。
3. **对话** —— 输入一句低动力测试语，比如「我明天有考试，但现在很累，开始不了」。
   - 预期：角色先温和回应 → 系统识别"低动力/难以开始" → 给出一个极小的具体行动（不是甩一长串待办）→ 语气始终守人设。
4. **10 分钟行动陪伴** —— 开始一次短专注，角色全程给鼓励/提醒，结束做 check-in。
5. **记忆摘要** —— 查看它记住了什么（偏好、设定、上次互动）。重点是体现"它不是一次性聊天工具"，且记忆可解释、可删除。
6. **3D 桌面设备** —— 见第 5 节。

## 5. 3D 桌面设备：不是概念，是已验证的品类

3D 桌面设备是 Yorimi 区别于所有纯软件 AI 陪伴产品的核心，也是最容易被低估的一块——它不是渲染概念或效果图，而是一台已经做出来、能摆上桌的实物。下面把它的技术路线、商用先例和实测参数一次讲清。

**技术路线。** Yorimi 的桌面设备走 **Pepper's Ghost 反射式全息** 路线：一块高亮度屏幕把角色画面投到倾斜的透明介质上，形成悬浮在设备内的立体影像，配独立驱动板做循环播放，不依赖外接电脑。这是一条成熟、可量产的光学方案，不是实验室概念。

**这条路线已经有商用产品在卖，证明它真实可行：**

- **Gatebox（日本）** —— 虚拟角色陪伴设备的代表性产品之一，证明“角色进入现实桌面空间”不是纯概念方向。参考：[Gatebox 官方网站](https://www.gatebox.ai/)。

- **AI HoloBox** —— 桌面级“全息显示 + AI 语音交互”设备，官网将其描述为 ChatGPT-powered holographic desktop companion，定位与 Yorimi 的桌面在场方向高度相关。参考：[AI HoloBox](https://www.aiholobox.com/)。

- **市场层面** —— 据 Global Market Insights，全球全息显示市场 2025 年规模约 43 亿美元，预计 2026–2035 年 CAGR 为 25.5%。参考：[GMInsights Holographic Display Market](https://www.gminsights.com/industry-analysis/holographic-display-market)。

一句话：Yorimi 不是在赌一个没人验证过的形态，而是在一个已有商用产品、已有付费用户、还在高速增长的品类里，做「原创 IP + 长期记忆 + 学习陪伴」的差异化打法。

**Yorimi 现有设备（已制作实物 Demo）：**

| 项 | 参数 |
| --- | --- |
| 显示屏 | 9.7 英寸 IPS 高亮度屏（约 24.1 × 18.5 cm，相当于第 4 代 iPad 屏幕大小），保证桌面环境与漫展强光下角色依然清晰 |
| 成像方式 | Pepper's Ghost 反射式全息：屏幕画面经倾斜透明介质反射，角色以立体形态悬浮在设备内 |
| 播放方式 | 内置驱动板独立循环播放角色内容，开机即用，无需外接电脑 |
| 展示内容 | 角色的表情、动作与状态（待机、对话、提醒、陪学习等场景的切换） |
| 形态定位 | 桌面摆件级别，可直接放在书桌/工位上，适合路演、漫展与日常陪伴展示 |

这台设备已经能在现场让观众直接看到"角色就在桌上、有表情、有状态"，是 Yorimi 与纯软件竞品最直观的区分点。

## 6. 常见问题

**装依赖报错** —— 删掉重装：`rm -rf node_modules package-lock.json && npm install`。

**端口被占** —— 修改 `.env` 里的 `PORT=3017`，例如改成 `PORT=3018`，再重启 `npm run dev`。

**对话没返回** —— 检查 `.env` 是否存在、`OPENAI_API_KEY` 是否正确、网络能否访问 OpenAI、额度是否够；临时没 Key 会进入 schema-compatible mock fallback。

**本地正常、线上部署失败** —— 多半是部署平台没配环境变量，或有本地存在但没提交进仓库的文件。

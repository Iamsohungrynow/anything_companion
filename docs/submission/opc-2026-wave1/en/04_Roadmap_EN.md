# Yorimi Roadmap: Wave 2 / Wave 3 / Wave 4

Current stage: Wave 1 Idea / Specs  
Track: Digital Culture, AI+Entertainment / AI+Education

## 1. Current Position

Wave 1 is about clarity: application scenarios, business logic, product requirements, and evaluation criteria.

Yorimi has:

- Product positioning.
- Chinese and English Wave 1 documentation.
- A deployed Web runtime prototype from the predecessor project.
- A physical 3D desktop device demo.
- Technical architecture.
- Evaluation plan.
- Early recognition from the UCWS Singapore Hackathon 2026 Top 20 via the predecessor project Compagnon Éveil.

## 2. Overall Roadmap

```text
Wave 2: Prototype
  Build a runnable Yorimi MVP from the current runtime and specs.

Wave 3: Validation
  Test with real users and creators, collect usage and willingness-to-pay signals.

Wave 4: Launch / Pitch
  Present a complete demo, validation data, creator pilots, and commercialization plan.
```

## 3. Wave 2: Prototype

Core loop:

```text
Character setup -> AI conversation -> low-motivation recognition
-> gentle response -> small action -> 10-minute sprint
-> check-in -> memory summary -> desktop display
```

P0:

- Yorimi-branded home and entry flow.
- Character creation with presets for study companion, original ACG character, and digital pet.
- Persona-consistent AI dialogue.
- Low-motivation study path.
- Check-in flow.
- Memory summary.
- Device display explanation and demo.

P1:

- Basic voice output.
- State-driven expression tags.
- Creator Studio upload mockup.
- 60-90 second demo video.

P2:

- Multi-character skins.
- Persistent memory.
- Voice input.
- Real-time device bridge.
- Subscription/waitlist prototype.

## 6. Business Model

- Consumer subscription: longer dialogue, better memory, premium voice, multi-character use.
- Content market: skins, voice packs, expressions, limited story interactions.
- Creator Studio: creator onboarding, analytics, revenue sharing, custom partnerships.
- 3D desktop device: device sales, creator collaborations, device skins, event demos, limited presales.

## 7. Risks and Responses

| Risk | Response |
| --- | --- |
| Positioning too broad | Lead with AI character interaction + 3D desktop presence; keep Creator Studio as expansion |
| Technical overload | Wave 2 only builds MVP; memory and device sync start lightweight |
| Copyright/IP risk | Accept only original or authorized characters; add review and takedown flow |
| Emotional dependency | Position as daily companionship and study support, not therapy or relationship replacement |
| Hardware overreach | Use hardware for demo and validation before any manufacturing plan |

## 8. Vision

Yorimi aims to become an AI character presence ecosystem connecting users, virtual characters, creators, and desktop devices. Users get companions with memory and presence. Creators turn characters into interactive services. Virtual characters move from one-time content into long-term digital partners.
